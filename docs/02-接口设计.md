# 供应商管理系统接口设计

## 1. 认证接口

### 1.1 SSO 登录

- `POST /api/auth/sso-login`
- 请求：

```json
{
  "ticket": "SSO-xxxx"
}
```

- 响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 1,
    "username": "admin",
    "realName": "系统管理员",
    "deptId": 1,
    "roleCodes": ["SYSTEM_ADMIN"],
    "permissionCodes": ["supplier:supplier:view"],
    "dataScope": 5,
    "accessToken": "xxx"
  }
}
```

### 1.2 当前登录用户

- `GET /api/auth/current-user`

## 3. 供应商管理接口

### 3.1 供应商列表

- `GET /api/suppliers`
- 查询参数：
  - `keyword`
  - `status`
  - `supplierType`

### 3.2 供应商详情

- `GET /api/suppliers/{id}`

### 3.3 新增供应商

- `POST /api/suppliers`
- 已落地字段：
  - `supplierCode`
  - `supplierName`
  - `supplierType`
  - `creditCode`
  - `legalPerson`
  - `contactPerson`
  - `contactPhone`
  - `contactEmail`
  - `province`
  - `city`
  - `address`
  - `bankName`
  - `bankAccount`
  - `taxpayerType`
  - `cooperationLevel`
  - `remark`

### 3.4 编辑供应商

- `PUT /api/suppliers/{id}`

### 3.5 准入审批

- `POST /api/suppliers/{id}/approve`

### 3.6 冻结 / 解冻

- `POST /api/suppliers/{id}/freeze`
- `POST /api/suppliers/{id}/unfreeze`
- 已落地

### 3.7 加入 / 移出黑名单

- `POST /api/suppliers/{id}/blacklist`
- `POST /api/suppliers/{id}/remove-blacklist`
- 已落地

## 7. 询价比价接口

- `GET /api/inquiries`
- `POST /api/inquiries`
- `GET /api/inquiries/{id}`
- `POST /api/inquiries/{id}/quotes`
- `GET /api/inquiries/{id}/compare`
- `GET /api/inquiries/{id}/export`

当前已落地接口：

- `GET /api/inquiries`
- `POST /api/inquiries`
- `POST /api/inquiries/{id}/compare`
- `GET /api/quotes?inquiryId={inquiryId}`
- `GET /api/quote-items?inquiryId={inquiryId}&quoteId={quoteId}`
- `POST /api/quotes`
- `POST /api/quotes/recommend/{inquiryId}`

补充说明：

- `POST /api/quotes` 已支持随报价单一起提交 `items` 行项目数组
- `POST /api/inquiries/{id}/compare` 当前已升级为多因子比价
  - 价格权重：40%
  - 交期权重：25%
  - 质保权重：15%
  - 历史评价权重：20%

## 13. 数据权限说明

- 当前已先落地到采购需求列表接口
  - 系统管理员 / 全部数据范围：查看全部
  - 部门数据范围：仅查看本部门 `deptId`
  - 个人数据范围：仅查看本人 `applicantId`
- 当前已扩展到以下接口
  - `GET /api/purchase-requests`
  - `GET /api/inquiries`
  - `GET /api/contracts`
  - `GET /api/operation-logs`
- 当前字段约定
  - 采购需求：`deptId` / `applicantId`
  - 询价单：`createdDeptId` / `createdBy`
  - 合同：`createdDeptId` / `createdBy`
  - 操作日志：`operatorDeptId` / `operatorId`
