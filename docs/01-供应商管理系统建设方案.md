# 供应商管理系统建设方案

## 1. 整体技术方案

### 1.1 系统定位

- 本系统是统一门户登录后的独立业务系统
- 不作为统一门户左侧内部菜单模块存在
- 用户通过统一门户应用入口跳转进入
- 系统自动识别当前用户身份，无需二次登录
- 复用统一门户的用户、角色、权限、应用访问控制和 SSO 认证能力

### 1.2 技术选型

- 前端：Vue 3 + TypeScript + Element Plus + Pinia + Vue Router
- 后端：Spring Boot 3 + MyBatis Plus + MySQL 8
- 文件：本地存储，预留 MinIO 扩展
- 鉴权：统一门户 SSO ticket / exchange / validate
- 日志：本地操作日志 + 门户统一口径兼容
- 缓存：Redis 预留，MVP 先以内存和数据库为主

### 1.3 单点登录方案

1. 用户在统一门户完成登录
2. 用户点击“供应商管理系统”应用卡片
3. 统一门户生成 `supplier` 应用的一次性 `sso_ticket`
4. 浏览器跳转到供应商系统入口并附带 `sso_ticket`
5. 供应商系统后端调用统一门户 `/sso/exchange`
6. 再调用统一门户 `/sso/validate` 获取用户、角色、权限、数据范围
7. 供应商系统签发本系统访问令牌或建立会话
8. 用户进入供应商系统首页

- 系统管理员

- `supplier:contract:view`
- `supplier:performance:view`
- `supplier:evaluation:view`
- `supplier:risk:view`
- `supplier:stats:view`

### 1.5 页面与交互原则

- 浅色系企业后台风格
- 独立左侧菜单、顶部栏、首页工作台
- 支持标签页导航
- 支持 KeepAlive 页面缓存
- 页面切换后保留查询条件、分页、滚动位置、表单草稿

- `sup_supplier_product_price_hist` 历史价格表
- `sup_purchase_req` 采购需求表
- `sup_purchase_req_item` 采购需求明细表
- `sup_inquiry` 询价单表
- `sup_quote` 报价表
- `sup_quote_item` 报价明细表
